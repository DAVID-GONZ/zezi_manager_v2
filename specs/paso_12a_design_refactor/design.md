## 4. `sync_tokens.py` — opción A (recomendada)

```python
"""
scripts/sync_tokens.py — regenera src/interface/design/tokens.py
parseando styles/tokens.css.

Uso:
    python scripts/sync_tokens.py            # regenera tokens.py
    python scripts/sync_tokens.py --check    # exit 1 si tokens.py difiere
"""
import re, sys
from pathlib import Path

TOKENS_CSS = Path("src/interface/design/styles/tokens.css")
TOKENS_PY  = Path("src/interface/design/tokens.py")

VAR_RE = re.compile(r"--([a-z0-9-]+)\s*:\s*([^;]+);")

# Mapeo nombre CSS → nombre Python (clase.atributo)
MAPPING = {
    "color-primary":          ("Colors", "PRIMARY"),
    "color-primary-dark":     ("Colors", "PRIMARY_DARK"),
    # ... lista completa (~80 entradas)
    "sidebar-width":          ("Layout", "SIDEBAR_WIDTH_PX"),  # se exporta como int
    "topbar-height":          ("Layout", "TOPBAR_HEIGHT"),
    "space-xs":               ("Spacing", "XS"),
    # ...
}

def parse_css() -> dict[str, str]:
    text = TOKENS_CSS.read_text(encoding="utf-8")
    return {m.group(1): m.group(2).strip() for m in VAR_RE.finditer(text)}

def render_py(values: dict[str, str]) -> str:
    # Construye el contenido completo de tokens.py
    ...

def main(check: bool = False) -> int:
    values = parse_css()
    generated = render_py(values)
    current = TOKENS_PY.read_text(encoding="utf-8") if TOKENS_PY.exists() else ""
    if check:
        if generated != current:
            print("DRIFT: tokens.py no coincide con tokens.css")
            print("Ejecuta: python scripts/sync_tokens.py")
            return 1
        return 0
    TOKENS_PY.write_text(generated, encoding="utf-8")
    print(f"Regenerado {TOKENS_PY} ({len(values)} variables)")
    return 0

if __name__ == "__main__":
    sys.exit(main(check="--check" in sys.argv))
```

### Notas de implementación

- Las clases `AsistenciaColors`, `DesempenoColors`, `Icons` **NO** se generan
  automáticamente: contienen métodos (`para_estado`, `css_badge`) y nombres de
  iconos que no viven en CSS. Se mantienen escritos a mano en `tokens.py`.
- El generador escribe la sección "Colors", "Spacing" y "Layout" como bloques bien
  delimitados, y respeta lo demás. Usa marcadores `# >>> AUTOGEN START` y
  `# <<< AUTOGEN END` para hacer el reemplazo robusto.

## 5. Migración de llamadores de `app_layout`

Buscar:
```
grep -r "app_layout(" src/interface/pages/ --include="*.py"
```

Cada llamada con firma vieja (positional, `titulo_pagina=...`) se reescribe a la
nueva firma:

```python
# Antes
app_layout(
    titulo_pagina="X",
    usuario_nombre=ctx.usuario_nombre,
    usuario_rol=ctx.usuario_rol,
    ruta_activa="/y",
    contenido=contenido,
    ctx=ctx,

# Después
app_layout(
    ctx,
    contenido,
    page_titulo="X",
    page_icono="...",      # si la página tenía page_header

```

Una vez migrados los callers, el bloque `_topbar_legacy` y la rama `if` que decide
se eliminan del `layout.py`.

## 7. Resolución del drift Layout.SIDEBAR_WIDTH

Decisión: 220 px gana (es el valor visible hoy).

- `tokens.css`: `--sidebar-width: 220px;`
- `tokens.py` (autogenerado): `Layout.SIDEBAR_WIDTH = 220`
- `styles/layout/sidebar.css`: usar `var(--sidebar-width)` consistentemente.

## 8. Alternativa descartada

**Migrar a CSS Modules / vanilla-extract.** Descartada: NiceGUI inyecta CSS plano,
no hay build step en el proyecto, añadir uno multiplica complejidad. Los módulos
por carpetas + concatenación cubren 100% de los beneficios reales aquí.
